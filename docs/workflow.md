# Development Workflow

## Recommended sequence

For reliable bring-up, always verify in volatile SRAM before flashing:

1. **Build** — synthesise, implement, generate bitstream
2. **Program** — load into FPGA SRAM (lost on power-off); verify on hardware
3. **Flash** — write to configuration memory (survives power cycle)

## VS Code tasks

| Task | Shortcut | What it does |
|------|----------|--------------|
| Build FPGA Design | `Ctrl+Shift+B` | Synthesis + implementation + bitstream |
| Program FPGA | `Ctrl+Shift+P` > Run Task | Write bitstream to FPGA SRAM (volatile) |
| Flash FPGA (Non-Volatile) | Run Task | Write bitstream to QSPI flash |
| Build and Program FPGA | Run Task | Build then program |
| Build and Flash FPGA | Run Task | Build then flash |
| Clean Build | Run Task | Delete the build directory |
| Configure Project (Setup GUI) | Run Task | Open the setup GUI |

## PowerShell commands

```powershell
.\scripts\build.ps1 -Action build        # synthesise and implement
.\scripts\build.ps1 -Action program      # program FPGA (volatile)
.\scripts\build.ps1 -Action flash        # flash configuration memory
.\scripts\build.ps1 -Action all          # build + program
.\scripts\build.ps1 -Action allflash     # build + flash
.\scripts\build.ps1 -Action clean        # delete build directory

# Override Vivado path for this run only
.\scripts\build.ps1 -Action build -VivadoPath "C:\Xilinx\Vivado\2023.2\bin\vivado.bat"
```

## Direct Vivado (batch mode)

```powershell
vivado -mode batch -source scripts/build.tcl
vivado -mode batch -source scripts/program.tcl
vivado -mode batch -source scripts/flash.tcl
```

## Build output

After a successful build, Vivado writes reports to the build directory (`C:/fpga_build` by default):

| File | Contents |
|------|----------|
| `timing_summary.rpt` | Setup/hold slack, worst path |
| `utilization.rpt` | LUT, FF, BRAM, DSP usage |
| `power.rpt` | Static and dynamic power estimate |
| `<project>.runs/impl_1/<top>.bit` | Bitstream |
| `<project>.runs/impl_1/<top>.mcs` | Flash image (generated by flash action) |

## Switching projects

Edit `scripts/config.tcl` (or use the setup GUI's Project Config tab):

```tcl
set PROJECT_NAME "new_project"
set TOP_MODULE   "new_top"
set SOURCE_FILES [list "new_top.v"]
set CONSTRAINT_FILES [list "new_project.xdc"]
```

Run `Clean Build`, then `Build FPGA Design`.

## Simulation

```powershell
.\scripts\build.ps1 -Action simulate
```

This uses `scripts/simulate.tcl`, which currently targets `reaction_game_tb`. Edit that file to point to your testbench.

Place testbenches in `testbench/` and source files in `src_main/`.
